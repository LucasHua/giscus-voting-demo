How Voting Works

1

CI Syncs Discussions

sync-vote-discussions.js creates one GitHub Discussion per plugin. Runs every 15 min via CI.

2

Cards Show Real Counts

CI exports reaction counts to vote-counts.json. Cards display read-only 👍👎 tallies from shared data.

3

Vote on Discussion

"💬 Vote on Discussion" opens the GitHub Discussion page. Users react with 👍 👎 ❤️ — counts refresh on next deploy.

4

Giscus Inline Voting

On detail pages, Giscus embeds the Discussion inline. Users react and comment without leaving the marketplace.

🔧 Setup for Repo Owner

  1. Enable Discussions on the repo (Settings → Features)
  2. Allow discussions: write in GitHub Actions permissions
  3. (Optional) Install Giscus GitHub App for inline Discussion embeds on detail pages
  4. Merge PR #14

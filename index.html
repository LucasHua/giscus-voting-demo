How It Works

1

CI Syncs Discussions

sync-vote-discussions.js creates one Discussion per plugin. Runs on deploy (every 15 min).

2

Cards Show Real Counts

CI exports reaction counts to vote-counts.json. Cards display read-only 👍👎 tallies.

3

Users Vote via Discussion

"Vote on Discussion" link opens the GitHub Discussion. Users react with 👍 👎 ❤️ — counts update on next deploy.

4

Giscus for Inline Voting

On detail pages, Giscus embeds the Discussion inline. Users react and comment without leaving the marketplace.

🔧 Setup for Repo Owner

To enable this on the production marketplace:

  1. Enable Discussions on the repo (Settings → Features)
  2. Allow discussions: write in GitHub Actions permissions
  3. (Optional) Install Giscus app for inline Discussion embeds
  4. Merge PR #14
